## Credential Proxy

| Feature | Status | Notes |
|---------|--------|-------|
| `--credential SERVICE` (reverse proxy injection) | Degraded | Proxy works; child not port-locked to proxy |
| `--env-credential` (env var injection) | Full | No proxy needed |
| `--env-credential-map` | Full | No proxy needed |
| `--network-profile` (host allowlist via proxy) | Degraded | Proxy works; no port lockdown |
| `--allow-domain` (domain allowlist) | Degraded | Proxy works; no port lockdown |
| `--allow-endpoint` (L7 method+path filtering) | Degraded | Filtering works at proxy; child can bypass proxy |
| `--upstream-proxy` (chain through external proxy) | Degraded | Proxy works; no port lockdown |
| `--upstream-bypass` | Degraded | Same as above |
| `--proxy-port` (fixed proxy port) | Degraded | Proxy binds correctly; no enforcement |
| Phantom token authentication | Full | Session token in env vars |
| Credential zeroization in memory | Full | Rust `Zeroizing<String>` |
| SSRF protection (cloud metadata blocking) | Full | Applied at proxy level |

**Why degraded:** On native Linux with Landlock V4, the child can *only* connect to `127.0.0.1:{proxy_port}`. On WSL2 with V3, there's no port-level filtering — the child could bypass the proxy and connect directly. The proxy itself works correctly. When Landlock V4 arrives, enforcement activates automatically.

### Root causes of all limitations

| Root Cause | Features Affected | Fix |
|------------|-------------------|-----|
| Landlock V4 missing (kernel 6.6 < 6.7) | Per-port network (3) | WSL2 kernel upgrade (automatic) |
| Landlock V6 missing (kernel 6.6 < 6.12) | signal_mode, process_info_mode, ipc_mode (3) | WSL2 kernel upgrade (automatic) |
| Landlock V5 missing (kernel 6.6 < 6.10) | Device ioctl filtering (1) | WSL2 kernel upgrade (automatic) |
| seccomp notify EBUSY | capability_elevation, interactive mode, PTY relay, runtime expansion (4) | microsoft/WSL#9548 fix or eBPF alternative |
| D-Bus `org.freedesktop.secrets` missing | trust keygen/sign/sign-policy/export-key (4) | Install gnome-keyring or keepassxc |
| Proxy port enforcement (Landlock V4) | All credential proxy features (6 degraded) | WSL2 kernel upgrade (automatic) |
